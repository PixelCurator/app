# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in PixelCurator, please report it
**privately** by email to **yves.vogl@mac.com**. Do not open a public issue for
security-sensitive problems.

Please include:

- a description of the vulnerability and its potential impact,
- step-by-step instructions to reproduce it, and
- any relevant logs, screenshots, or proof-of-concept code.

## Response and Disclosure

You can expect an initial acknowledgement within **5 business days**. We aim to
investigate and provide a remediation plan or fix within **30 days** of a valid
report, depending on severity and complexity. We will keep you informed of
progress and coordinate responsible disclosure once a fix is available.

## Supported Versions

PixelCurator is under active development. Security fixes are applied to the
latest `main` branch and the current released build.
